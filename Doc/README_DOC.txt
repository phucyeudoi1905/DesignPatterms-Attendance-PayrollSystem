📋 HƯỚNG DẪN

Thư mục này dành cho BÁO CÁO ĐỒ ÁN (file Word).

Vui lòng thêm file báo cáo của bạn vào đây theo format:

   📄 [MSSV]_[HoTen]_BaoCaoDoAn.docx

Ví dụ:
   📄 12345678_NguyenVanA_BaoCaoDoAn.docx

📝 NỘI DUNG BÁO CÁO NÊN BAO GỒM

1. Trang bìa
   - Tên trường
   - Tên đồ án
   - Thông tin sinh viên
   - Thông tin giảng viên
   - Ngày nộp

2. Mục lục

3. Giới thiệu
   - Mục đích đồ án
   - Phạm vi
   - Công nghệ sử dụng

4. Phân tích yêu cầu
   - Use cases
   - Functional requirements
   - Non-functional requirements

5. Thiết kế hệ thống
   - Kiến trúc tổng quan
   - Database schema
   - Class diagram
   - Sequence diagram

6. Design Patterns (QUAN TRỌNG!)
   
   6.1. Strategy Pattern
   - Mô tả
   - Class diagram
   - Code minh họa
   - Lợi ích khi áp dụng
   
   6.2. Singleton Pattern
   - Mô tả
   - Class diagram
   - Code minh họa
   - Lợi ích khi áp dụng
   
   6.3. Decorator Pattern
   - Mô tả
   - Class diagram
   - Code minh họa
   - Lợi ích khi áp dụng

7. Cài đặt và triển khai
   - Hướng dẫn cài đặt
   - Cấu hình môi trường
   - Hướng dẫn chạy ứng dụng

8. Kết quả
   - Screenshots ứng dụng
   - Demo các tính năng
   - Test cases

9. Kết luận
   - Đánh giá
   - Hạn chế
   - Hướng phát triển

10. Tài liệu tham khảo

💡 GỢI Ý

- Đánh giá ưu/nhược điểm
